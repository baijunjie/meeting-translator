# iOS native integration — `RealtimeAsr` plugin

On-device ASR for the iOS (Capacitor) app: **Silero VAD + SenseVoice** via
[k2-fsa/sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx), captured with `AVAudioEngine`.

This guide is for a developer on **macOS with Xcode + CocoaPods**. The native iOS
project already exists at `apps/ios/ios/App/App.xcworkspace` (generated by Capacitor 7).
The steps below were **NOT executed/built here** (no iOS Simulator runtime was available
when this was written) — the on-device ASR is unverified until you run them.

What was written and is ready to wire in (`apps/ios/native-plugin/ios/`):

| File | Role |
| --- | --- |
| `RealtimeAsrPlugin.swift` | The plugin: mic capture → 16 kHz mono Float32 → Silero VAD → SenseVoice; emits `partial`/`segment`/`status`/`setupProgress`; `start`/`stop`/`getSetupStatus`/`downloadModels`/`getMicStatus`/`openMicSettings`. **Fully implemented.** |
| 注册方式 | 插件实现 `CAPBridgedPlugin`（`identifier`/`jsName`/`pluginMethods`）。Capacitor 7 只自动注册 pod 插件，本插件编在 App target，故由 `ios/App/App/MainViewController.swift`（`CAPBridgeViewController` 子类）在 `capacitorDidLoad()` 调 `bridge?.registerPluginInstance(...)` 手动注册。已弃用旧的 `.m` / `CAP_PLUGIN` 宏。 |
| `SherpaOnnx.swift` | The **official sherpa-onnx Swift wrapper**, vendored verbatim from tag `v1.13.3` (Apache-2.0). Wraps the C API as `SherpaOnnxOfflineRecognizer`, `SherpaOnnxVoiceActivityDetectorWrapper`, etc. |
| `App-Bridging-Header.h` | ObjC→Swift bridging header; `#import "sherpa-onnx/c-api/c-api.h"` (also vendored from `v1.13.3`). |
| `AsrModels.swift` | **Generated** from `@rt/core`'s model registry (`pnpm --filter @rt/ios gen:models`). Model URLs / filenames / dirs / "is-complete" checklist — same source of truth as macOS. Do not edit by hand. |

The TS side is done and type-checks: `apps/ios/src/bridge.ts` (the `AppBridge` impl)
and `apps/ios/native-plugin/{definitions,index}.ts` (the JS plugin contract).

## 0. Prerequisites

- macOS + Xcode (with command line tools), CocoaPods (`sudo gem install cocoapods`).
- Node + pnpm; run `pnpm install` at the repo root first.
- An Apple developer signing identity. The Simulator is fine for UI and for SenseVoice
  recognition (CPU); **microphone capture is best validated on a real device.**

## 1. Build the web bundle + sync Capacitor

```bash
pnpm --filter @rt/ios build      # → apps/ios/dist/
pnpm --filter @rt/ios cap:sync   # = `cap sync ios`: copies dist/ + installs Capacitor pods
```

`@capacitor/preferences` is already a dependency, so `cap sync` wires its pod automatically.
(App icons, optional: `pnpm --filter @rt/ios icons`.)

## 2. Regenerate the model registry Swift file (only if the registry changed)

`apps/ios/native-plugin/ios/AsrModels.swift` is generated from `@rt/core`
(`packages/core/src/models.ts`) so iOS and macOS never drift. It is committed, so you
normally don't need to run this — but if you change the registry:

```bash
pnpm --filter @rt/ios gen:models           # rewrite AsrModels.swift
pnpm --filter @rt/ios gen:models --check    # CI: fail if it's stale
```

## 3. Add the sherpa-onnx xcframeworks to the App target

sherpa-onnx ships **prebuilt iOS xcframeworks** as a GitHub release artifact — no need to
build from source. There is **no official SwiftPM package** (only community forks, e.g.
`willwade/sherpa-onnx-spm`); the official, version-matched path is the xcframework + the
vendored `SherpaOnnx.swift` wrapper, which is what this plugin uses.

**Version pinned: `v1.13.3`** (onnxruntime 1.26.0). The vendored `SherpaOnnx.swift` +
`App-Bridging-Header.h` were copied from this exact tag — **if you bump the xcframework,
re-copy both from the matching tag** (`swift-api-examples/SherpaOnnx.swift` and
`swift-api-examples/SherpaOnnx-Bridging-Header.h`), since the Swift wrapper must match the
bundled C API.

```bash
cd apps/ios/ios/App        # next to App.xcworkspace
mkdir -p Frameworks && cd Frameworks
# 41 MB no-TTS build — we only need ASR + VAD. (Use sherpa-onnx-v1.13.3-ios.tar.bz2 if you also want TTS.)
curl -L -o sherpa-ios.tar.bz2 \
  https://github.com/k2-fsa/sherpa-onnx/releases/download/v1.13.3/sherpa-onnx-v1.13.3-ios-no-tts.tar.bz2
tar xjf sherpa-ios.tar.bz2
# Yields ./build-ios/ containing:
#   build-ios/sherpa-onnx.xcframework                       (static lib: ios-arm64 + ios-arm64_x86_64-simulator)
#   build-ios/ios-onnxruntime/1.26.0/onnxruntime.xcframework (dynamic framework: same two slices)
mv build-ios/sherpa-onnx.xcframework .
mv build-ios/ios-onnxruntime/1.26.0/onnxruntime.xcframework .
rm -rf build-ios sherpa-ios.tar.bz2
```

In **Xcode** (open `App.xcworkspace`), select the **App** target → **General** →
**Frameworks, Libraries, and Embedded Content** → **+** → **Add Other… → Add Files…** and
add both:

- `Frameworks/sherpa-onnx.xcframework` — static lib; set to **Do Not Embed**.
- `Frameworks/onnxruntime.xcframework` — dynamic framework; set to **Embed & Sign**.

> Both xcframeworks ship a device slice (`ios-arm64`) **and** a simulator slice
> (`ios-arm64_x86_64-simulator`), so Simulator and device builds both link cleanly.

## 4. Add the plugin + wrapper sources to the App target

本仓库已完成接线（提交在 `apps/ios/ios/`），无需手动操作；以下供了解/维护：

- 规范源在 `native-plugin/ios/`，以**原地引用**方式（相对路径，非拷贝）加入 App target：
  `RealtimeAsrPlugin.swift` / `SherpaOnnx.swift` / `AsrModels.swift`，加上 app 壳层的
  `ios/App/App/MainViewController.swift`。改这些文件无需再同步副本。
- 桥接头让 `SherpaOnnx.swift` 看见 sherpa-onnx C API：
  `SWIFT_OBJC_BRIDGING_HEADER = $(PROJECT_DIR)/../../native-plugin/ios/App-Bridging-Header.h`；
  `HEADER_SEARCH_PATHS` 含 `$(PROJECT_DIR)/Frameworks/sherpa-onnx.xcframework/Headers`。
- 注册：`MainViewController`（`CAPBridgeViewController` 子类）在 `capacitorDidLoad()` 调
  `bridge?.registerPluginInstance(RealtimeAsrPlugin())`，storyboard 已指向该子类。
  插件 id / 方法名在 `index.ts`、`definitions.ts` 与 `RealtimeAsrPlugin.swift` 的
  `CAPBridgedPlugin`（`jsName = "RealtimeAsr"` + `pluginMethods`）三处保持一致。

## 5. Microphone permission

Already added to `apps/ios/ios/App/App/Info.plist`:

```xml
<key>NSMicrophoneUsageDescription</key>
<string>Realtime Translator uses the microphone to transcribe and translate speech on your device.</string>
```

(Capacitor does not inject arbitrary Info.plist privacy strings from `capacitor.config.ts`,
so this must live in the native Info.plist — which it now does. `cap sync` does not
overwrite it.) The plugin requests permission in `start()` (`AVAudioApplication`/
`AVAudioSession requestRecordPermission`), reports status via `getMicStatus()`, and
`openMicSettings()` opens the app's iOS Settings page.

## 6. Models — download on first run (implemented)

The app **downloads the models on first run** (no large binaries committed / bundled).
`getSetupStatus()` reports whether they're present; `downloadModels()` fetches them with
`URLSession` (following GitHub/HF redirects), reporting byte progress via the
`setupProgress` event. They land in the app sandbox at:

```
<App>/Library/Application Support/models/
  silero_vad.onnx                                                  (~2.2 MB)
  sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2024-07-17/tokens.txt    (~0.3 MB)
  sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2024-07-17/model.int8.onnx (~228 MB)
```

URLs/filenames/dirs come from `AsrModels.swift` (generated from `@rt/core`), identical to
what macOS downloads. The TS bridge already routes `downloadAsrModels()` → plugin
`downloadModels()` and forwards `setupProgress` → `onSetupProgress`.

**Optional — bundle instead of download** (to skip the ~230 MB first-run download): add the
three files to the App target as resources, then change `modelsDir()` in
`RealtimeAsrPlugin.swift` to resolve from `Bundle.main` (and `modelsReady()` accordingly).
Downloading is the default to keep the app binary small.

> The ~228 MB `model.int8.onnx` download can outlive a foreground session. The current
> implementation uses a default (non-background) `URLSession`; for resilience against app
> backgrounding mid-download, switch to a background `URLSessionConfiguration` later.

## 7. Build & run

Once the Simulator runtime is installed (or on a device):

```bash
# After any web change: rebuild + copy into the native project.
pnpm --filter @rt/ios build && pnpm --filter @rt/ios cap:sync

# Open Xcode and run (Cmd-R) on a simulator or device:
pnpm --filter @rt/ios exec cap open ios

# Or fully from the CLI (example — adjust scheme/destination):
xcodebuild -workspace apps/ios/ios/App/App.xcworkspace -scheme App \
  -destination 'platform=iOS Simulator,name=iPhone 15' build
```

Iterate: edit web → `build` + `cap:sync`; edit native → re-`cp` the changed files (step 4)
and build in Xcode.

### First-run flow to expect

1. UI calls `getSetupStatus()` → `asrReady:false` → shows the download screen.
2. `downloadAsrModels()` → plugin downloads (~230 MB), `setupProgress` drives the bar.
3. `startPipeline()` → mic permission prompt → `status:loading` (model load + warm-up) →
   `status:running`. Speak → `partial` updates live, `segment` on each pause.

## The sherpa-onnx Swift API used (for reference)

From the vendored `SherpaOnnx.swift` (`v1.13.3`):

```swift
// SenseVoice offline recognizer
let featConfig = sherpaOnnxFeatureConfig(sampleRate: 16000, featureDim: 80)
let senseVoice = sherpaOnnxOfflineSenseVoiceModelConfig(
  model: "<…>/model.int8.onnx", language: "", useInverseTextNormalization: true)
let modelConfig = sherpaOnnxOfflineModelConfig(
  tokens: "<…>/tokens.txt", numThreads: 2, debug: 0, senseVoice: senseVoice)
var cfg = sherpaOnnxOfflineRecognizerConfig(featConfig: featConfig, modelConfig: modelConfig)
let recognizer = SherpaOnnxOfflineRecognizer(config: &cfg)
let result = recognizer.decode(samples: floats, sampleRate: 16000)   // result.text, result.lang

// Silero VAD
let silero = sherpaOnnxSileroVadModelConfig(model: "<…>/silero_vad.onnx",
  threshold: 0.35, minSilenceDuration: 0.35, minSpeechDuration: 0.25,
  windowSize: 512, maxSpeechDuration: 7.0)
var vcfg = sherpaOnnxVadModelConfig(sileroVad: silero, sampleRate: 16000, numThreads: 1, debug: 0)
let vad = SherpaOnnxVoiceActivityDetectorWrapper(config: &vcfg, buffer_size_in_seconds: 120)
vad.acceptWaveform(samples: window512)              // feed fixed 512-sample windows
vad.isSpeechDetected()                              // live speech flag (drives partials)
while !vad.isEmpty() { let s = vad.front(); vad.pop(); /* s.samples, s.start */ }
vad.flush()                                         // on stop, drain trailing speech
```

`SenseVoice result.lang` is a short code (`zh`/`en`/`ja`/`yue`/`ko`); the plugin defensively
strips any `<|…|>` wrapping so its output matches macOS.

## Platform notes / what differs from macOS

- **Audio capture is native, not `getUserMedia`.** The `RealtimeAsr` plugin captures the mic
  with `AVAudioEngine` and runs the whole pipeline natively; the TS bridge does not send
  audio. (`public/audio-worklet.js` is kept for parity but does not feed the recognizer.)

- **Translation is cloud-only (for now).** `apps/ios/src/bridge.ts` runs translation in JS via
  `@rt/core`'s `CloudTranslator`. Selecting the local `m2m100` engine emits a
  `translation:status` `error` — on-device translation is deferred on iOS (no
  onnxruntime-mobile translator yet). Future work: a `RealtimeTranslate` native plugin
  mirroring `RealtimeAsr`, or a Core ML conversion.

- **Storage** uses `@capacitor/preferences` (async KV) for settings + archives, reusing the
  platform-agnostic pure logic in `@rt/core`.

## Pipeline behavior vs the macOS reference

Mirrors `apps/macos/src/main/pipeline.ts` intent: 16 kHz mono, **partial while speaking**
(best-effort, throttled to ~0.6 s, re-decoding the in-progress speech buffer) + **final on
segment end**. Segmentation is delegated to sherpa-onnx's Silero VAD (`minSilenceDuration`
0.35 s for snappy finals; `maxSpeechDuration` 7 s so a non-stop talker still gets periodic
finals) rather than the bespoke energy-minimum splitter macOS uses — a reasonable first cut.
Text post-processing (strip CJK inter-character spaces, collapse repeated-token ASR
hallucinations, drop punctuation-only segments) is ported from macOS's `cleanAsrText`.
